Video #3

Front End Legos: Better Design with Reusable HTML & CSS

Shay Howe

Link to Front End Legos: Better Design with Resuable HTML & CSS

Shay Howe starts off his video presentation with talking about how coding at times can seem a little uneasy like on a ship. Coding is fragile and difficult at time, not always expecting the unexpected. A lot has to go into coding to build and create something new. Howe goes on to talk about bad practices to avoid when it comes to code and especially coding CSS. Howe talks about how it is bad to avoid classes. Especially when you have a lot going on in your code you have to be able to tell which is which and what goes with what. I found this useful because I have this problem. Classes, divs and making ids do scare me at time. I never know or have complete trust in putting them in the correct places or if I then code them right. Next, Howe talks about the organizations off your CSS are very important. He organizes his CSS on Base, component, and modules. This I found interest because I never thought about how learning to organize your organizations will help to make things run more smoothly and look great. Howe says he doesn’t want coding CSS to feel like a job and in fact wants it to feel as he is the captain with full control over what he is putting out. I completely agree because the more confortable you get with coding, the easier it is to tackle and have complete control. Lastly what I found interesting goes back to the classes and ids. Naming the classes is crucial and important. You want to make sure the name you give it can’t be found anywhere else on your code and at the same time it is distinct enough. This is useful for people like me, who do fear the classes and ids.
